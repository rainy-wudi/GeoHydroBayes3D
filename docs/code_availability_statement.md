**Hardware requirements:**  
The Python workflow can be executed on a standard personal computer or workstation. A multi-core CPU and at least 8 GB RAM are recommended for the demonstration workflow. GPU acceleration is optional for PA-DNN surrogate-model training when a compatible PyTorch environment is available. For the MATLAB-based random-field and Abaqus input-file generation workflow, 16 GB RAM or higher is recommended for large finite-element meshes and element-wise permeability mapping.

**Program language:**  
Python 3.10 or later; MATLAB.

**Software required:**  
Python 3.10+, Conda or pip, NumPy, pandas, SciPy, scikit-learn, matplotlib, PyYAML, joblib, and PyTorch. The emcee package is optional for affine-invariant ensemble MCMC sampling; if emcee is unavailable, the code falls back to a simple random-walk sampler. MATLAB is required for the random-field generation scripts. The Statistics and Machine Learning Toolbox is recommended for Latin Hypercube Sampling; if unavailable, the repository includes a simplified LHS fallback implementation. The Parallel Computing Toolbox is optional for accelerating batch generation. Abaqus is required to run the generated `.inp` files.

**Program size:**  
The public source-code package is lightweight because large generated Abaqus files and third-party software environments are excluded. The program source code is approximately 4.72 MB (4.72 MB counted as text/source files in this release) excluding Python/MATLAB/Abaqus installations and generated simulation outputs. The actual installed environment size depends on the local Python distribution and third-party packages, especially PyTorch. Abaqus output files generated by users may be substantially larger depending on the mesh size and number of random-field realizations.
